### Panduan OJS 3.3: Langkah Strategis Merespons Kiriman Naskah (Modul 4\)

#### Pendahuluan: Peran Sentral Editor dalam Alur Kerja OJS 3.3

Dalam ekosistem Open Journal Systems (OJS) 3.3, Editor memiliki otoritas tertinggi untuk memastikan integritas dan kelancaran alur kerja editorial. Editor tidak hanya memantau naskah, tetapi juga memiliki kemampuan teknis untuk mengelola akun pengguna, login sebagai pengguna lain jika diperlukan, dan mengambil keputusan krusial pada setiap tahap. Tahap Kiriman ( *Submission* ) adalah pintu gerbang utama; di sinilah Editor bertindak sebagai kurator awal yang menentukan apakah sebuah naskah layak diteruskan ke tahap peninjauan sejawat ( *peer review* ).Fleksibilitas OJS 3.3 memungkinkan sistem ini beradaptasi dengan berbagai skala organisasi jurnal. Sebuah jurnal dapat dikelola secara efisien oleh Editor tunggal yang menangani seluruh proses, atau oleh tim besar yang terdiri dari Editor Kepala ( *Editor-in-Chief* ) sebagai pengawas, beberapa Editor Bagian ( *Section Editor* ) yang bertanggung jawab atas rubrik spesifik, hingga staf pendukung seperti Editor Copyedit dan Layout.

#### Memahami Dasbor Editorial: Navigasi dan Strategi Monitoring

Dasbor editorial adalah pusat kendali bagi Editor. Untuk manajemen jurnal yang proaktif, Editor harus memahami fungsi empat tab utama:

* **My Queue:**  Menampilkan naskah yang menjadi tanggung jawab langsung Anda. Fitur ini krusial untuk menjaga akuntabilitas kerja dalam tim editorial yang besar.  
* **Unassigned:**  Tab ini adalah  **area risiko utama** . Naskah yang berada di sini belum memiliki Editor yang ditugaskan. Sebagai konsultan, saya menekankan pentingnya memantau tab ini secara rutin untuk mencegah naskah "terlantar", terutama jika ada Editor yang mengundurkan diri atau berhalangan di tengah jalan.  
* **All Active:**  Menyediakan visibilitas penuh terhadap seluruh naskah yang sedang diproses di semua tahapan (Submission, Review, Copyediting, Production). Editor Kepala sering menggunakan tab ini untuk memantau beban kerja tim.  
* **Archives:**  Rekam jejak permanen naskah yang telah dipublikasikan atau ditolak.Guna meningkatkan efisiensi navigasi, manfaatkan fitur pencarian (berdasarkan nama penulis atau kata kunci judul) serta filter spesifik. OJS 3.3 menyediakan filter canggih berdasarkan status terlambat ( *overdue* ), naskah belum lengkap ( *incomplete* ), tahap proses, bagian jurnal, hingga filter  **Days since last activity**  yang sangat berguna untuk mengidentifikasi naskah yang stagnan.

#### Bedah Detail Rekam Naskah (Submission Record)

Klik tombol  **"View"**  untuk masuk ke rincian naskah. Di bagian atas halaman, Editor dapat mengakses alat manajemen data penting:

* **Header Rekam Naskah:**  Informasi cepat mencakup ID Naskah, nama belakang Penulis, dan judul.  
* **Activity Log vs. Internal Notes:**  Penting untuk membedakan keduanya.  **Activity Log**  adalah rekaman otomatis yang tidak dapat diubah ( *immutable* ) mengenai setiap tindakan sistem. Sementara itu, tab  **Notes**  digunakan untuk komunikasi manual internal antar-Editor guna mencatat hal-hal strategis terkait naskah.  
* **Submission Library:**  Tempat berbagi dokumen pendukung (seperti formulir etik atau data tambahan) secara efisien dengan penulis dan tim editorial tanpa melalui prosedur pengiriman formal.

#### Manajemen File Kiriman dan Komponen Naskah

Area  **"Submission Files"**  menampilkan aset digital naskah. Dalam OJS 3.3, pengelolaan file harus memperhatikan  **Komponen Naskah**  ( *Submission Components* ) seperti  *Article Text, Research Materials, Transcripts,*  atau  *Source Text* .Tindakan yang dapat dilakukan Editor:

1. **Mengunduh:**  Klik judul file untuk mengunduh.  
2. **Pro-Tip (Expansion Arrow):**  Klik tanda panah biru kecil di sebelah file untuk mengakses log aktivitas spesifik file dan catatan internal terkait file tersebut.  
3. **Manajemen File:**  Anda dapat mengedit nama file, menghapusnya, atau menggunakan tombol  **"Upload File"**  untuk menambahkan dokumen baru.  
4. **Download All Files:**  Mengambil seluruh paket dokumen kiriman dalam satu kali klik untuk peninjauan  *offline* .

#### Optimalisasi Fitur Diskusi (Pre-review Discussions)

Fitur diskusi adalah kunci agar komunikasi editorial tetap terpusat dan terlacak di dalam sistem, bukan tersebar di email pribadi.**Instruksi Penggunaan Discussion Wizard:**

1. Klik  **"Add Discussion"** .  
2. Pilih partisipan (Penulis atau Editor lain).  
3. Masukkan subjek dan pesan. Anda juga dapat melampirkan file di sini.  
4. Klik "OK".Selain itu, Editor dapat menggunakan fitur  **"Notify"**  pada daftar partisipan di sisi kanan layar. Fitur ini memungkinkan pengiriman pesan menggunakan templat email yang tersedia. Gunakan diskusi ini secara strategis untuk meminta klarifikasi naskah atau perbaikan administratif sebelum memutuskan untuk mengirim naskah ke tahap  *review* .

#### Tiga Keputusan Editorial Utama (Decision Wizard)

Setelah tinjauan awal selesai, Editor harus mengeksekusi keputusan menggunakan  *Decision Wizard*  yang tersedia di panel sebelah kanan:| Tindakan | Fungsi dan Skenario Penggunaan || \------ | \------ || **Send to Review** | Memindahkan naskah ke tahap peninjauan.  **Catatan Strategis:**  Pastikan Anda memilih file yang tepat. Uncheck (hapus centang) pada file yang mengandung identitas penulis untuk menjaga integritas  *double-blind review* . || **Accept and Skip Review** | Digunakan untuk kiriman yang tidak memerlukan  *peer review*  formal, seperti komentar editorial atau pengumuman, agar langsung masuk ke tahap Copyediting. || **Decline Submission** | Menolak naskah yang tidak sesuai standar atau lingkup. Anda dapat memilih untuk mengirim notifikasi atau hanya merekam keputusan. Jika terjadi kesalahan, gunakan fitur  **"Change Decision"**  untuk membatalkan status penolakan. |

#### Penugasan Editor Bagian (Section Editor)

Jika jurnal menggunakan model tim, Editor Kepala dapat menugaskan Editor Bagian melalui tombol  **"Assign"**  yang akan membuka  *Assignment Wizard* .Dalam tahap ini, terdapat tiga kontrol perizinan kritis yang harus diatur:

1. **Hanya Rekomendasi:**  Editor Bagian hanya bisa memberikan saran, tidak bisa mengambil keputusan final atau menghubungi penulis secara mandiri.  
2. **Wewenang Penuh:**  Editor Bagian memiliki kendali penuh atas alur naskah hingga pengambilan keputusan.  
3. **Metadata Permission:**  Anda dapat membatasi hak akses Editor Bagian untuk mengubah metadata naskah. Ini adalah fitur krusial untuk menjaga integritas data bibliografi jurnal.Setelah penugasan dikonfirmasi, sistem akan mengirimkan email otomatis kepada Editor Bagian dan memperbarui daftar partisipan secara  *real-time* .

#### Kesimpulan dan Langkah Berikutnya

Penguasaan terhadap Modul 4 ini memastikan bahwa setiap naskah yang masuk ke jurnal Anda dikelola dengan standar profesionalitas yang tinggi. Dengan memanfaatkan dasbor secara optimal, menjaga integritas file kiriman, dan mengatur penugasan dengan izin yang tepat, Editor dapat mempercepat waktu respons awal jurnal.Langkah strategis selanjutnya setelah naskah dinyatakan layak adalah memilih pakar yang tepat untuk mengevaluasi kualitas substansi. Hal ini akan dibahas secara mendalam pada  **Modul 5: Penugasan Reviewer** .
